Microsoft Linux 没来，但微软用更聪明的办法赢了

Microsoft Linux 没来，但微软用更聪明的办法赢了

老文新评 #062 · ZDNet{rel="nofollow noopener"} · 2020-10-12

2020 年 10 月，TechRepublic 的 Jack Wallen 写了一篇相当激进的长文——"微软应该直接发布自己的 Linux 桌面"，不是 WSL、不是 Linux 子系统，而是 OEM 出厂预装的完整 Linux 发行版，名字都帮你想好了：Microsoft Linux。

6 年后看这事，他猜对了一半——微软确实在 WSL、Azure、VS Code、PowerShell、Edge 这些层面深度拥抱了 Linux。但 OEM 出厂预装 Microsoft Linux？没发生。微软选了另一条路。

2020 年的微软处于一个微妙的转型期：Azure 已经是云市场老二、Windows 桌面市场开始被 Chromebook 和 Mac 蚕食、WSL 已经迭代到第二代（带真 Linux 内核）、VS Code 成为最流行的跨平台编辑器。那一年也是疫情爆发期，远程办公突然成为主流，Linux 桌面首次真正进入主流企业视野。Wallen 这篇文章正是在这个时间窗口提出「微软应该 all-in Linux」——结合 Eric Raymond 当时那条「桌面战争终战」的论点。

文章讲了啥

Wallen 的核心论点可以拆成几条：

  • Azure 是新摇钱树，Windows 不是：微软已经在云上赢了，没必要为 Windows 桌面续命
  • 微软应该"登月式飞跃"：不是"在 Linux 内核上跑 Windows 界面"，而是直接发布完整的 Microsoft Linux 发行版
  • OEM 全切换：大卖场 PC 出厂直接装 Microsoft Linux，没有"半途而废的余地"
  • 保留 Windows Defender：用户需要防钓鱼/恶意 URL，Defender 应该原生集成到 Linux 版
  • 保留 Microsoft 365：通过 SaaS 提供，用户感受不到 Windows 没了
  • 桌面环境二选一或定制：GNOME/KDE/Pantheon/Xfce 任选，UI 可以做成"看起来像 Windows"

核心收益：用户得到更稳定更安全的桌面、IT 不用天天修 Windows 漏洞、Linux 社区终于主导桌面。

这些观点今天还成立吗？

当时的判断 现在 (2026) 评分
微软会发完整 Linux 桌面发行版 没发生，微软选了"嵌入 Linux 到 Windows"路线 ❌
Azure 才是微软的未来 Azure 现在是云市场第二，Office 365 + Azure 主导营收 ✅
OEM 出厂装 Linux 能主导桌面 Steam Deck 证明了 Linux 桌面可行，但 OEM 没跟 ⚠️
Windows Defender 应该上 Linux Microsoft Defender for Linux 已存在，但定位不同 ⚠️
用户能接受"看起来像 Windows 的 Linux" WSL + Windows 11 实现了类似体验 ⚠️

现在 (2026) 再读这篇文章

Wallen 错在他假设微软会做"激进的战略转向"，但微软选了更聪明的渐进路线——把 Linux 嵌入 Windows（WSL）、把 Windows 服务搬到 Linux（SQL Server、Edge、Defender）、把 Linux 体验带给所有人（VS Code、GitHub Codespaces）。

OEM 没出厂预装 Microsoft Linux，但Microsoft Surface 现在默认带 WSL，Windows 11 也能跑 Linux GUI 应用。从用户体验角度，"我既能跑 Windows 软件又能用 Linux 工具"这个目标已经达成，只是实现路径不同。

更重要的是，Wallen 当年没看到的变量：AI Agent 时代到来。2026 年的开发者并不在意「用 Windows 还是 Linux」，他们在乎「AI Agent 能不能跑在本地」。微软押对的是 GitHub Copilot + VS Code + Azure AI 这条路，不是操作系统本身。

Wallen 2020 年提出「Microsoft Linux」——路径没押对，但 Azure + WSL + VS Code 这套「嵌入 Linux」路线微软走得更好。
